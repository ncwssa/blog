## 1. 项目概述

### 1.1 项目定位

本项目是一款面向个人开发者的**本地知识总结博客系统**，旨在帮助用户以 Markdown 形式记录、整理、检索和总结个人学习笔记与技术知识。系统集成 AI 能力，支持语义搜索与智能总结，打造个人专属的知识管理与复习平台。

### 1.2 项目目标

- **知识积累**：提供便捷的 Markdown 编辑环境，帮助用户高效记录学习笔记
- **知识检索**：基于 RAG（检索增强生成）技术，支持语义级别的知识搜索
- **智能总结**：借助多厂商大模型 API，对知识进行 AI 总结与问答
- **能力提升**：作为全栈练手项目，涵盖前端、后端、数据库、AI 集成等技术栈
- **开源分享**：代码结构清晰，适合在 GitHub 上展示与分享

### 1.3 项目特色

| 特色 | 说明 |
|------|------|
| 本地部署 | 无需服务器，启动即用，数据完全自主掌控 |
| 无需登录 | 单用户本地系统，省去鉴权流程，简化使用体验 |
| AI 赋能 | 集成多厂商大模型，支持语义搜索、智能总结、知识问答 |
| 多模型切换 | 灵活配置 OpenAI、智谱AI、通义千问、DeepSeek 等模型 |
| 数据可迁移 | 支持数据导出/导入，保障数据安全与可移植性 |
| Monorepo | 前后端代码集中管理，一键启动，便于维护与分享 |

## 3. 功能需求

### 3.1 博客管理

博客是系统的核心内容单元，用户通过博客记录学习笔记。

#### 3.1.1 新建博客

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标题 | 文本 | 是 | 博客标题，最大长度 200 字符 |
| 分类 | 选择 | 是 | 从分类列表中选择，支持单选 |
| 内容 | Markdown | 是 | 博客正文，使用 Markdown 编辑器 |
| 标签 | 文本 | 否 | 可选标签，逗号分隔 |

- 提供 Markdown 实时预览
- 支持常用 Markdown 快捷键（加粗、标题、列表、代码块等）
- 保存时自动生成向量嵌入并存入向量数据库

#### 3.1.2 编辑博客

- 加载已有博客内容到编辑器
- 支持修改标题、分类、内容、标签
- 保存时更新向量数据库中对应记录

#### 3.1.3 删除博客

- 提供删除确认弹窗
- 删除后同步移除向量数据库中的对应记录
- 支持批量删除

#### 3.1.4 博客列表

- 分页展示所有博客
- 支持按分类筛选
- 支持按标题关键词搜索
- 支持按创建时间 / 更新时间排序
- 列表项显示：标题、分类、创建时间、摘要（前 100 字）

#### 3.1.5 博客详情

- 完整渲染 Markdown 内容
- 显示标题、分类、标签、创建时间、更新时间
- 提供「编辑」和「删除」操作入口
- 支持代码块语法高亮

---

### 3.2 分类系统

分类用于组织和归类博客内容，便于用户按知识领域管理笔记。

#### 3.2.1 预设分类

系统初始化时预设以下常用分类：

| 序号 | 分类名称 | 说明 |
|------|---------|------|
| 1 | 前端 | HTML/CSS/JavaScript/框架等 |
| 2 | 后端 | Node.js/Java/Python/Go 等 |
| 3 | 算法 | 数据结构与算法 |
| 4 | 数据库 | SQL/NoSQL/数据库原理 |
| 5 | 网络 | HTTP/TCP/计算机网络 |
| 6 | 操作系统 | 进程/线程/内存管理 |
| 7 | 工具 | Git/Docker/编辑器/效率工具 |
| 8 | 读书笔记 | 技术书籍读书笔记 |
| 9 | 面试准备 | 面试题整理与复习 |
| 10 | 项目总结 | 项目经验与复盘 |

#### 3.2.2 自定义分类

- 用户可新增自定义分类
- 支持编辑分类名称
- 支持删除分类（需先处理该分类下的博客：移至其他分类或一并删除）
- 分类名称不可重复

### 3.3 RAG 知识库

基于向量数据库实现检索增强生成（RAG），将博客内容转化为可语义检索的知识库。

#### 3.3.1 自动向量化

- 博客创建 / 更新时，自动将内容分块（chunk）并生成向量嵌入
- 分块策略：按段落或固定 token 长度切割，保留上下文
- 向量存储：使用 sqlite-vss 存储向量数据
- 向量维度：根据所选 Embedding 模型确定

#### 3.3.2 语义搜索

- 用户输入自然语言查询
- 系统将查询转为向量，在向量数据库中进行相似度检索
- 返回相关度最高的博客片段列表
- 展示匹配片段所属的博客标题和相关内容
- 支持设置检索数量（Top K）

#### 3.3.3 知识问答

- 基于语义搜索结果，将相关上下文注入 LLM Prompt
- 由 AI 生成基于个人知识库的回答
- 支持多轮对话
- 显示引用来源（对应的博客标题和片段）

---

### 3.4 AI 智能助手

基于 RAG（检索增强生成）技术，将所有博客内容构建为个人知识库，提供智能问答、博客总结和多轮对话能力。

#### 3.4.1 知识问答

- 用户输入任意自然语言问题
- 系统自动将问题向量化，从知识库中语义检索最相关的博客片段（Top-K）
- 将检索到的相关片段作为上下文，结合用户问题构造 Prompt，调用大模型生成答案
- AI 回答中标注引用来源（对应的博客标题），用户可点击跳转查看原文
- 当知识库中无相关内容时，AI 如实告知并提供通用性建议

#### 3.4.2 博客总结

- **单篇总结**：在博客详情页提供「AI 总结」按钮，调用大模型生成当前博客的结构化摘要
- **多篇总结**：用户选择多篇博客（或按分类选择），AI 生成跨文档的综合总结
- 支持自定义总结 Prompt（如：提取要点、生成思维导图大纲、面试知识点归纳等）
- 总结结果以弹窗或侧边栏形式展示，支持复制和导出
- 适用场景：面试前快速复习某个领域的全部笔记

#### 3.4.3 多轮对话

- 提供类似 ChatGPT 的对话界面，支持上下文连续追问
- AI 记住对话历史，理解追问中的指代和省略
- 每次提问自动触发 RAG 检索，确保回答基于最新知识库内容
- 支持创建多个对话，每个对话独立维护上下文
- 对话历史持久化存储，重新打开可继续之前的对话

#### 3.4.4 知识库检索展示

- AI 回答时展示检索到的相关博客片段列表
- 每个片段显示：来源博客标题、匹配片段内容、相关度评分
- 用户可点击片段查看完整博客原文
- 支持调整检索参数（Top-K 数量、相似度阈值）

---

### 3.5 多模型管理

支持配置和管理多个 AI 厂商的大模型 API。

#### 3.5.1 模型配置

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 厂商名称 | 选择 | 是 | OpenAI / 智谱AI / 通义千问 / DeepSeek |
| API Key | 文本 | 是 | 厂商提供的 API 密钥 |
| API Base URL | 文本 | 否 | 自定义接口地址（用于代理或私有部署） |
| 模型名称 | 文本 | 是 | 如 gpt-4o、glm-4、qwen-turbo、deepseek-chat |
| 备注 | 文本 | 否 | 用户自定义备注 |

#### 3.5.2 模型管理功能

- **添加配置**：填写上述字段，保存模型配置
- **编辑配置**：修改已有模型的配置信息
- **删除配置**：移除不需要的模型配置
- **切换模型**：设置当前使用的默认模型
- **连接测试**：验证 API Key 和接口地址是否可用

#### 3.5.3 支持厂商

| 厂商 | 接口格式 | 典型模型 |
|------|---------|---------|
| OpenAI | OpenAI API | gpt-4o, gpt-4o-mini |
| 智谱AI | OpenAI 兼容 | glm-4, glm-4-flash |
| 通义千问 | OpenAI 兼容 | qwen-turbo, qwen-plus |
| DeepSeek | OpenAI 兼容 | deepseek-chat, deepseek-reasoner |

---

### 3.6 数据迁移

支持数据的导出与导入，保障数据安全和可移植性。

#### 3.6.1 数据导出

- 导出格式：JSON 文件
- 导出内容包括：
  - 所有博客数据（标题、内容、分类、标签、时间戳）
  - 分类数据
  - 模型配置（API Key 可选脱敏导出）
- 支持全量导出
- 支持按分类导出

#### 3.6.2 数据导入

- 导入 JSON 文件恢复数据
- 导入策略：
  - 覆盖模式：清空现有数据后导入
  - 合并模式：与现有数据合并（冲突时以导入数据为准）
- 导入后自动重建向量索引
- 导入前显示数据预览和冲突提示

## 5. 用户流程

### 5.1 系统启动流程

```
克隆仓库 → 安装依赖(pnpm install) → 启动开发服务器(pnpm dev) → 浏览器打开系统
```

### 5.2 写博客流程

```
首页 → 点击「新建博客」→ 输入标题 → 选择分类 → 编写 Markdown 内容
→ 点击「保存」→ 系统自动向量化存储 → 跳转到博客详情页
```

### 5.3 搜索知识流程

```
首页/知识库页面 → 输入自然语言问题 → 系统语义检索
→ 展示相关博客片段列表 → 点击查看完整博客
```

### 5.4 AI 问答流程

```
进入 AI 对话页面 → 输入问题 → 系统检索相关知识
→ 构建 Prompt（问题 + 知识上下文）→ 调用大模型 → 展示 AI 回答 + 引用来源
```

### 5.5 AI 总结流程

```
博客详情页 → 点击「AI 总结」→ 调用大模型生成总结 → 展示总结内容

或：

博客列表 → 勾选多篇博客 → 点击「批量总结」→ AI 生成综合总结 → 展示总结结果
```

### 5.6 模型配置流程

```
设置页面 → 模型管理 → 点击「添加模型」→ 选择厂商
→ 填写 API Key 和模型名称 → 测试连接 → 保存 → 设为默认模型
```

### 5.7 数据迁移流程

```
导出：设置页面 → 数据管理 → 点击「导出数据」→ 选择导出范围 → 下载 JSON 文件

导入：设置页面 → 数据管理 → 点击「导入数据」→ 上传 JSON 文件
→ 预览数据 → 选择导入策略 → 确认导入 → 重建向量索引
```

### 6.3 页面详细描述

#### 首页/仪表盘

- 博客总数、分类数量等统计卡片

- 快捷入口：新建博客、搜索知识、AI 对话
- 各分类博客数量分布

#### 博客列表页

- 顶部：搜索框 + 分类筛选 + 排序选择
- 主体：博客卡片/列表视图
- 底部：分页控件
- 操作：支持批量选择进行删除或 AI 总结

#### 博客编辑页

- 左栏：Markdown 编辑器（带工具栏）
- 右栏：实时预览
- 顶部：标题输入、分类选择、标签输入
- 底部：保存按钮

#### 知识库搜索页

- 搜索输入框（支持自然语言）
- 搜索结果列表：相关片段 + 相关度评分 + 来源博客
- 点击结果跳转到对应博客

#### AI 对话页（AI 智能助手）

- 左侧：对话列表侧边栏，显示所有历史对话，支持新建/删除对话
- 右侧：当前对话内容区，消息气泡形式展示用户/AI 交替对话
- 底部：输入框 + 发送按钮，支持 Enter 快捷发送
- AI 回答中标注引用来源（博客标题），可点击查看原文
- 回答下方展示检索到的相关知识片段列表
- 支持流式输出，实时显示 AI 生成内容
- 整体布局类似 ChatGPT，简洁直观

#### 设置页

- 模型管理：模型配置列表 + 添加/编辑/删除/测试
- 数据管理：导出按钮 + 导入上传区域
