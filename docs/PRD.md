## 1. 项目概述

### 1.1 项目定位

本项目是一款面向个人开发者的**本地知识总结博客系统**，旨在帮助用户以 Markdown 形式记录、整理、检索和总结个人学习笔记与技术知识。系统集成 AI 能力，支持语义搜索与智能总结，打造个人专属的知识管理与复习平台。

### 1.2 项目目标

- **知识积累**：提供便捷的 Markdown 编辑环境，帮助用户高效记录学习笔记
- **知识检索**：基于 RAG（检索增强生成）技术，支持语义级别的知识搜索
- **智能总结**：借助多厂商大模型 API，对知识进行 AI 总结与问答
- **能力提升**：作为全栈练手项目，涵盖前端、后端、数据库、AI 集成等技术栈
- **开源分享**：代码结构清晰，适合在 GitHub 上展示与分享

### 1.3 项目特色

| 特色 | 说明 |
|------|------|
| 本地部署 | 无需服务器，启动即用，数据完全自主掌控 |
| 无需登录 | 单用户本地系统，省去鉴权流程，简化使用体验 |
| AI 赋能 | 集成多厂商大模型，支持语义搜索、智能总结、知识问答 |
| 多模型切换 | 灵活配置 OpenAI、智谱AI、通义千问、DeepSeek 等模型 |
| 数据可迁移 | 支持数据导出/导入，保障数据安全与可移植性 |
| Monorepo | 前后端代码集中管理，一键启动，便于维护与分享 |

## 3. 功能需求

### 3.1 博客管理

博客是系统的核心内容单元，用户通过博客记录学习笔记。

#### 3.1.1 新建博客

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标题 | 文本 | 是 | 博客标题，最大长度 200 字符 |
| 分类 | 选择 | 是 | 从分类列表中选择，支持单选 |
| 内容 | Markdown | 是 | 博客正文，使用 Markdown 编辑器 |
| 标签 | 文本 | 否 | 可选标签，逗号分隔 |

- 提供 Markdown 实时预览
- 支持常用 Markdown 快捷键（加粗、标题、列表、代码块等）
- 保存时自动生成向量嵌入并存入向量数据库

#### 3.1.2 编辑博客

- 加载已有博客内容到编辑器
- 支持修改标题、分类、内容、标签
- 保存时更新向量数据库中对应记录

#### 3.1.3 删除博客

- 提供删除确认弹窗
- 删除后同步移除向量数据库中的对应记录
- 支持批量删除

#### 3.1.4 博客列表

- 分页展示所有博客
- 支持按分类筛选
- 支持按标题关键词搜索
- 支持按创建时间 / 更新时间排序
- 列表项显示：标题、分类、创建时间、摘要（前 100 字）

#### 3.1.5 博客详情

- 完整渲染 Markdown 内容
- 显示标题、分类、标签、创建时间、更新时间
- 提供「编辑」和「删除」操作入口
- 支持代码块语法高亮

### 3.3 RAG 知识库

基于检索增强生成（RAG）技术，将博客内容转化为可语义检索的知识库。

#### 3.3.1 自动向量化

- 博客创建 / 更新时，自动将内容分块（chunk）并生成向量嵌入（不阻塞响应）
- **分块策略**：按 Markdown 标题（`##`、`###`）优先分割，递归降级到段落、行、句号，最大块 512 tokens，相邻块重叠 50 tokens
- 预处理：自动去除 YAML frontmatter 元信息
- **向量存储**：Embedding 以 JSON 数组形式存储在 `post_chunks.embedding` 列中
- 向量维度：由所选 Embedding 模型决定（如 text-embedding-3-small 为 1536 维）

#### 3.3.2 语义搜索

- 用户输入自然语言查询，系统将查询转为向量
- 在 JS 中计算余弦相似度，返回相关度最高的文本片段
- 展示匹配片段、来源博客标题和相似度评分
- 支持设置检索数量（Top K）和相似度阈值

#### 3.3.3 三级降级策略

系统设计了三层渐进降级，确保在任何条件下都能返回搜索结果：

| 优先级 | 条件 | 检索方式 |
|:---:|------|---------|
| **1** | 有向量数据 + Embedding API 正常 | 余弦相似度语义搜索 |
| **2** | 有分块但无向量 | `chunk_text LIKE` 关键词匹配 |
| **3** | 从未索引过 | `title LIKE` 标题搜索 |

降级时前端会显示黄色提示条，引导用户配置支持 Embedding 的 AI 模型。

### 3.4 AI 智能助手（未实现 — 后续版本）

> **注：本节功能计划在后续版本中实现，当前阶段仅完成知识库搜索。**

基于 RAG（检索增强生成）技术，提供智能问答、博客总结和多轮对话能力。

#### 3.4.1 知识问答

- 用户输入任意自然语言问题
- 系统自动将问题向量化，从知识库中语义检索最相关的博客片段（Top-K）
- 将检索到的相关片段作为上下文，结合用户问题构造 Prompt，调用大模型生成答案
- AI 回答中标注引用来源（对应的博客标题），用户可点击跳转查看原文

#### 3.4.2 博客总结

- **单篇总结**：在博客详情页提供「AI 总结」按钮，调用大模型生成当前博客的结构化摘要
- **多篇总结**：用户选择多篇博客（或按分类选择），AI 生成跨文档的综合总结

#### 3.4.3 多轮对话

- 提供类似 ChatGPT 的对话界面，支持上下文连续追问
- AI 记住对话历史，理解追问中的指代和省略
- 每次提问自动触发 RAG 检索，确保回答基于最新知识库内容
- 支持创建多个对话，每个对话独立维护上下文

---

### 3.5 多模型管理

支持配置和管理多个 AI 厂商的大模型 API。

#### 3.5.1 模型配置

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 厂商名称 | 选择 | 是 | OpenAI / 智谱AI / 通义千问 / DeepSeek |
| API Key | 文本 | 是 | 厂商提供的 API 密钥 |
| API Base URL | 文本 | 否 | 自定义接口地址（用于代理或私有部署） |
| 模型名称 | 文本 | 是 | 如 gpt-4o、glm-4、qwen-turbo、deepseek-chat |
| 是否启用 | 开关 | 否 | 是否启用该模型 |

#### 3.5.2 模型管理功能

- **添加配置**：填写上述字段，保存模型配置
- **编辑配置**：修改已有模型的配置信息（API Key 编辑时需重新输入）
- **删除配置**：移除不需要的模型配置（使用确认弹窗）
- **设为默认**：将某模型设为默认模型，系统优先使用
- **启用/禁用**：控制模型是否参与 Embedding 生成
- **连接测试**：向模型发送一条测试消息，验证 API Key 和接口地址是否可用
- **重建索引**：配置完 Embedding 模型后一键重建知识库向量索引

#### 3.5.3 支持厂商

| 厂商 | 接口格式 | 典型模型 | 支持 Embedding |
|------|---------|---------|:------------:|
| OpenAI | OpenAI API | gpt-4o, gpt-4o-mini | ✅ |
| 智谱AI | OpenAI 兼容 | glm-4, glm-4-flash | ✅ |
| 通义千问 | OpenAI 兼容 | qwen-turbo, qwen-plus | ✅ |
| DeepSeek | OpenAI 兼容 | deepseek-chat, deepseek-reasoner | ❌ |

> **说明：** DeepSeek API 仅提供 Chat Completion 接口，不提供独立的 Embedding API，因此无法用于向量化。使用 DeepSeek 作为默认模型时，知识库搜索会自动降级为关键词匹配。

## 5. 用户流程

### 5.1 系统启动流程

```
克隆仓库 → 安装依赖(pnpm install) → 启动开发服务器(pnpm dev) → 浏览器打开系统
```

### 5.2 写博客流程

```
首页 → 点击「新建博客」→ 输入标题 → 选择分类 → 编写 Markdown 内容
→ 点击「保存」→ 系统自动触发向量化（异步）→ 跳转到博客详情页
```

### 5.3 搜索知识流程

```
知识库搜索页 → 输入自然语言问题 → 系统检测是否有向量数据
├─ 有向量 → 语义检索（余弦相似度）→ 展示匹配片段 + 评分
└─ 无向量 → 降级为关键词匹配 → 展示结果 + 黄色提示条
→ 点击结果查看完整博客
```

### 5.4 模型配置流程

```
设置页面 → AI 模型管理 → 点击「添加模型」→ 选择厂商
→ 填写 API Key 和模型名称 → 测试连接 → 保存
→ 点击「设为默认」→ 点击「重建索引」→ 系统遍历所有文章分块并向量化
```

### 5.5 重建索引流程

```
设置页面 → AI 模型管理 → 点击「重建索引」
→ 确认弹窗 → 系统逐篇处理 → 显示结果统计
  (已向量化 X 篇 / 仅分块 Y 篇 / 失败 Z 篇)
```
